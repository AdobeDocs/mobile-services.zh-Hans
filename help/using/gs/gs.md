# 入门指南{#getting-started}

此信息可帮助您了解和使用 Adobe Mobile Services。

Adobe Mobile Services 由以下部分组成：

* Adobe Mobile Services UI
* Adobe Mobile SDK

对于在提高用户参与度和证明移动设备应用程序投资回报率 (ROI) 方面寻求最有效方法的企业公司，Adobe 提供了端到端解决方案 Adobe Mobile Services，来获取和吸引应用程序用户，并分析和优化他们的体验。

如今，移动设备应用程序的现状与首次推出智能手机时截然不同。仅依靠移动设备应用程序来将客户与品牌连接起来是不够的；今天，您必须在各种渠道中打造的具有凝聚力和吸引力的客户体验，并将您的移动设备应用程序作为战略触点，吸引最忠诚的高价值客户。但要使这些用户与您的应用程序进行交互，您的应用程序需要具备引人注目的内容、上下文通知、智能个性化和集成应用程序分析等等。

## Adobe Mobile Services UI {#mobile-services-ui}

以下浏览器支持 Mobile Services UI：

* Google Chrome（最新的两个版本）
* Mozilla Firefox（最新的两个版本）
* Apple Safari（最新的两个版本）
* Microsoft Edge（最新的两个版本）

Adobe Mobile Services 可帮助通过列方式提高移动设备应用程序参与度：

### 获取

在&#x200B;*获取*&#x200B;中，您可以使用付费媒体、自有媒体和免费媒体推动领先应用商店中应用程序下载的用户获取。通过使用 Adobe Mobile Services，您可以加快应用程序用户的客户获取过程。

Adobe Mobile Services 提供用户客户获取工作流程，包括客户获取跟踪链接和深层链接，可以测量您的渠道在获取应用程序用户方面的有效性。通过跟踪哪些用户来自哪个渠道的营销链接，您可以了解哪个赢取渠道最有效地推动了盈利和吸引用户。

此外，通过深层链接，您可以将用户直接引导至您希望他们看到的应用程序内容，并鼓励他们在必要时安装您的应用程序。

客户获取具有以下主要功能：

* 适用于应用程序的客户获取分析
* 跨应用商店的跟踪链接
* 到应用程序的深层链接
* 回发与广告网络的集成

有关此阶段的更多信息，请参阅 [客户获取](/help/using/acquisition-main/acquisition-main.md).

### 分析

在&#x200B;*分析*&#x200B;中，您可以了解消费者使用移动设备应用程序的方式，以及促使他们转换或返回的内容。

通过 Adobe Analytics，您可以深入了解用户如何下载、安装和打开您的应用程序。您还可以测量和分析您的应用程序内容和 UI，执行队列分析、路径和流失。通过 Adobe Analytics，您可以使用中央数据存储来提供您的营销决策，并减少贵组织中的营销数据竖井。

您可以使用 Adobe Audience Manager 通过丰富的数据来扩充您的受众区段，以提供更多的上下文体验和个人体验。

*分析*&#x200B;具有以下主要功能：

* 应用程序参与度分析
* 路径和漏斗分析
* 队列和维系分析
* 位置分析
* 广泛的设备和平台支持

有关可以运行和分析客户的报表的更多信息，请参阅 [报表](/help/using/usage/usage.md).

### 参与

在&#x200B;*参与*&#x200B;中，您可以使用相关的推送通知和应用程序内消息与您的用户通信。通过定向推送通知和应用程序内消息，可确保用户会继续返回您的应用程序。通过对 Analytics 中区段的支持，您可以将推送通知定位到可以进行响应的用户区段，并提高其转换意向。

*参与*&#x200B;具有以下主要功能：

* 推送通知由分析区段触发。
* 应用程序内消息由实时分析、警报和新选件/内容触发。
* 了解查看次数、点进率和下游行为

### Adobe移动消息传递

您可以使用推送通知和应用程序内通知与您的用户进行通信。推送通知通过设备上的操作系统发送，而应用程序内消息在用户积极与应用程序进行交互时由应用程序发送。应用程序内消息通常可以包含其他一些格式，如弹出窗口和插播式广告。

在 Adobe Mobile 中，您可以配置以下类型的消息：

**推送通知**，在您的应用程序外部显示，提供以下功能：

* 通过相关的推送通知提高重新参与度。
* 创建、划分消息并将消息发送至已下载品牌应用程序并选择接收推送通知的客户。
* 服务器端由应用商店而不是从移动设备应用程序发送。

有关创建推送通知的更多信息，请参阅[创建推送消息](/help/using/in-app-messaging/t-create-push-message/t-create-push-message.md)。

**应用程序内通知**&#x200B;具有以下功能：

* 当用户在应用程序会话中时，驱动他们执行特定操作。
* 其他格式（警报、全屏），因为消息由应用程序传递，而不是由推送传递网络传递。
* 由实时分析触发。
* 允许交叉推广应用程序和产品。
* 鼓励用户对应用商店进行评级。
* 提供实时和基于位置的消息

有关创建应用程序内消息的更多信息，请参阅[创建应用程序内消息](/help/using/in-app-messaging/t-in-app-message/t-in-app-message.md)。

### 优化

在&#x200B;*优化*&#x200B;中，您可以优化转换（订阅、商业、广告收入等）并提高客户维系率。优化应用程序中的用户体验可帮助您个性化内容，从而实现最大的投资回报率和转化率。

有关测试和 Adobe Target 的更多信息，请转到 [Adobe Target](https://docs.adobe.com/content/help/en/target/using/target-home.html)。

### 地域定向

在移动设备中，您可以通过 GPS 位置基本了解消费者与您的应用程序交互或在后台运行应用程序时他们所在的位置。借助地域定向，Adobe Target 使您能够根据地理位置提供量身定制的相关内容、选件或消息。您可以定位属于目标点定义范围内的用户，或者位于 iBeacons 附近的用户以及使用相关推送通知的用户。

Adobe Target for Mobile 应用程序现在可充分利用 Adobe Analytics 提供的增强区段和报表。这意味着 Adobe Target 可以利用 Analytics 中的所有关键应用程序量度来定位和个性化；这也会有利于更深层次的测试报表取得成功，从而使营销人员可以更好地了解那些“假设”的问题 - 答案可以避免应用程序营销人员承受展示应用程序投资回报率的压力。适用于应用程序集成的 Analytics/Target 提供了一个组合产品，它代表了市场上最强大的应用程序参与解决方案。

有关位置的更多信息，请参阅以下内容：

* [Mobile Services用户指南中的位置](/help/using/location/c-location-overview.md)
* Android SDK 手册中的[位置](/help/android/location/location.md)
* iOS SDK 手册中的[位置](/help/ios/location/location.md)

## Adobe Mobile SDK {#mobile-services-sdk}

Adobe 提供了一个端到端的移动营销解决方案，可以提高所有这些领域的客户参与度。通过一个 SDK，您可以访问 Adobe Analytics、Adobe Campaign 和 Adobe Audience Manager 的功能，从而降低管理多个不同 SDK 的技术成本。

Adobe Mobile SDK 提供以下功能：

* 端到端的移动设备参与

   您可以使用一个集成的轻量级 Adobe Mobile SDK 跨平台测量和优化应用程序。
* 获取新客户，享受互动瞬间

   * 通过定向推送通知，包括对富媒体和应用程序内消息的支持，使用户反复返回您的应用程序。
   * 使用深层链接将应用程序用户直接引导至您想要他们查看的内容。

* 测量和优化体验以提高投资回报率

   您可以深入了解应用程序生命周期量度，包括漏斗分析（下载、安装、打开）、操作（包括会话时长、崩溃次数、信标和消息交互）。
* 全面

   * 广泛支持领先的移动操作系统和跨平台开发工具。
   * 智能手机、平板电脑、可穿戴设备和 OTT（顶级）控制台支持广泛的设备。

* 集成

   * 一个用于多种解决方案的 SDK（Analytics、Campaign 和 Audience Manager），可减少开发人员的实施时间和工作量。
   * 仅需一行代码即可收集“基线”应用程序生命周期量度。
   * 随着移动策略的改进，您可以轻松激活 Adobe Experience Cloud 功能来获取、分析和吸引用户。

* 快速且轻量

   * 最大限度地减少用于将数据发送到 Adobe 服务器和第三方系统的设备处理负载。
   * 占用空间小，可以减小提交给应用商店的应用程序包大小。

有关 Adobe Mobile SDK 的更多信息，请参阅[适用于 Experience Cloud 解决方案的 Android SDK 4.x](https://docs.adobe.com/content/help/en/mobile-services/android/overview.html) 和[适用于 Experience Cloud 解决方案的 iOS SDK 4.x](https://docs.adobe.com/content/help/en/mobile-services/ios/rel-notes.html)。
