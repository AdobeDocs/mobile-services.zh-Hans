# 入门指南{#getting-started}

此信息可帮助您了解和使用 Adobe Mobile Services。

AdobeMobile Services由以下部分组成：

* Adobe Mobile Services UI
* Adobe Mobile SDK

对于寻求提高用户参与度并证明其移动App投资投资回报(ROI)最有效方法的企业公司,Adobe提供端到端的解决方案，即Adobe移动服务，以赢取和吸引App用户并分析和优化其体验。

如今，移动应用的格局与智能手机首次启动时截然不同。 拥有移动应用程序连接客户与品牌已不够；如今，您必须在渠道间创造有凝聚力、引人入胜的客户体验，并将移动App作为战略切入点来吸引最忠诚、最有价值的客户。 但是，要让这些用户与您的App互动，需要引人注目的内容、情境式通知、智能个性化和集成的App分析等。

## Adobe Mobile Services UI {#mobile-services-ui}

以下浏览器支持Mobile Services UI:

* Google Chrome（最新的两个版本）
* Mozilla Firefox（最新两个版本）
* Apple Safari（最新的两个版本）
* Microsoft Edge（最新的两个版本）

Adobe Mobile Services 可帮助通过列方式提高移动设备应用程序参与度：

### 获取

在 *Acquire*&#x200B;中，您使用付费、自有和免费媒体推动用户赢取，以便在领先的应用商店内下载应用程序。 使用AdobeMobile Services，您可以加快应用程序用户获取过程。

AdobeMobile Services提供用户赢取工作流，包括赢取跟踪和深度链接，这些渠道衡量您赢取应用程序用户的效率。 借助可跟踪哪些用户来自哪些渠道的营销链接，您可以深入了解哪些客户获取渠道在吸引有利可图的用户方面最有效。

此外，通过深层链接，您可以将用户直接引导到您希望他们查看的应用程序内容中，并鼓励他们在必要时安装您的应用程序。

客户赢取优惠以下主要功能：

* 应用程序客户获取分析
* 跨应用商店跟踪链接
* 深层链接到应用程序
* 与广告网络的后置集成

有关此阶段的详细信息，请参 [阅客户获取](/help/using/acquisition-main/acquisition-main.md)。

### 分析

在“ *分析*”中，您可以了解消费者如何使用移动应用程序，以及他们转化或返回的原因。

通过Adobe Analytics，您可以深入了解用户下载、安装和打开应用程序的方式漏斗。 您还可以衡量和分析您的App内容和UI，进行同期群分析、寻路和流失。 借助Adobe Analytics，您可以使用中央数据存储来指导您的营销决策并减少组织内的营销数据孤岛。

您可以使用Adobe Audience Manager丰富受众细分，提供更符合情境的个性化体验。

*分析* 优惠以下主要功能：

* 应用互动分析
* 寻路和漏斗分析
* 同期群和保留分析
* 位置分析
* 广泛的设备和平台支持

有关可运行和分析客户的报表的更多信息，请参阅 [报表](/help/using/usage/usage.md)。

### 参与

在 *Engage*&#x200B;中，您可以使用相关推送通知和应用程序内消息与用户通信。 通过有针对性的推送通知和应用程序内消息传递，您可以确保用户继续返回您的应用程序。 借助对Analytics中区段的支持，您可以将推送通知目标到会做出响应并增加其转换倾向的用户区段。

*吸引* 优惠使用以下主要功能：

* 推送通知由分析区段触发。
* 应用程序内消息由实时分析、警报和新优惠/内容触发。
* 了解视图、点击率和下游行为

### Adobe Mobile 消息传送

您可以使用推送和应用程序内通知与用户进行通信。 推送通知通过设备上的操作系统发送，而当用户与应用程序进行主动交互时，应用程序内消息将在应用程序中发送。 应用程序内消息通常可包括各种其他格式，如弹出窗口和插播广告。

在Adobe移动中，您可以配置以下类型的消息：

**推送通知**（显示在应用程序外）优惠以下功能：

* 通过相关的推送通知推动重新参与。
* 创建、细分和发送消息给已下载品牌应用程序并通过选择加入接受的客户，以接收推送通知。
* 由应用程序商店而非移动应用程序在服务器端发送。

For more information about creating push notifications, see [Create a push message](/help/using/in-app-messaging/t-create-push-message/t-create-push-message.md).

**应用程序内通知** ,优惠以下功能：

* 在用户处于应用程序会话中时，将用户引导至特定操作。
* 其他格式（警报、全屏），因为消息是由应用程序而不是推送投放网络传送的。
* 由实时分析触发。
* 允许跨渠道推广应用程序和产品。
* 鼓励用户离开应用商店评级。
* 提供实时和基于位置的消息

有关创建应用程序内消息的详细信息，请 [参阅创建应用程序内消息](/help/using/in-app-messaging/t-in-app-message/t-in-app-message.md)。

### 优化

在“ *优化*”中，您可以优化转化率(订阅、商务、广告收入等)并提高客户保留率。 优化应用程序中的用户体验可以帮助您个性化内容，从而提高最高ROI和转化率。

有关测试和 Adobe Target 的更多信息，请转到 [Adobe Target](https://docs.adobe.com/content/help/zh-Hans/target/using/target-home.html)。

### 地域定向

移动设备使您能够固有地了解消费者在与您的应用程序互动时所处的位置，或者让其通过GPS位置在后台运行。 通过地理定位，Adobe Target使您能够在邻近性很重要的时候提供定制的、相关的内容、优惠或消息。 您可以目标位于某个目标点的已定义半径内或在他们靠近iBeacons时使用相关推送通知的用户。

Adobe Target移动应用程序现在充分利用了Adobe Analytics提供的增强细分和报告。 这意味着Adobe Target可以利用Analytics中的所有关键应用程序指标进行目标和个性化；它还使测试成功的报告程度更加深入，使营销人员能够更好地了解那些“如果”的问题——这些问题的答案可能会规避应用程序营销人员面临的显示应用程序投资回报的压力。 App集成的分析/目标可提供组合产品，这代表市场上可用的最强大的App互动解决方案。

有关位置的更多信息，请参阅以下内容：

* [Mobile Services 用户指南中的位置](/help/using/location/c-location-overview.md)
* Android SDK 手册中的[位置](/help/android/location/location.md)
* [iOS SDK 手册中的位置](/help/ios/location/location.md)

## Adobe Mobile SDK {#mobile-services-sdk}

Adobe 提供了一个端到端的移动营销解决方案，可以提高所有这些领域的客户参与度。使用一个SDK，您可以访问Adobe Analytics、Adobe Campaign和Adobe Audience Manager的功能，从而降低管理多个不同SDK的技术成本。

Adobe移动SDK优惠以下功能：

* 端对端移动互动

   您可以使用一个轻量、集成的Adobe移动SDK跨平台衡量和优化应用程序。
* 赢取新客户并交付引人入胜的微瞬间

   * 通过目标推送通知（包括富媒体支持和应用程序内消息），让用户一次又一次地重新访问您的应用程序。
   * 使用深层链接将应用程序用户直接带入您想要的内容。

* 衡量和优化体验以提高ROI

   您可以深入了解应用生命周期指标，包括漏斗分析（下载、安装、打开）、包括会话长度、崩溃、信标和消息传递交互等操作。
* 全面

   * 对领先的移动操作系统和跨平台开发工具的广泛支持。
   * 跨智能手机、平板电脑、可穿戴设备和OTT（自上而下）控制台提供广泛的设备支持。

* 集成

   * 一个用于多个解决方案(分析、活动和Audience Manager)的SDK，可减少开发人员的实施时间和工作。
   * 收集“基线”应用程序生命周期指标只需要一行代码。
   * 随着移动战略的发展，您可以轻松激活Adobe Experience Cloud的功能，以赢取、分析和吸引用户。

* 快速轻量

   * 将向Adobe服务器和第三方系统发送数据的设备处理负载降至最低。
   * 占用空间小会最小化提交到应用程序商店的应用程序包的大小。

For more information about the Adobe Mobile SDKs, see [Android SDK 4.x for Experience Solutions](https://docs.adobe.com/content/help/zh-Hans/mobile-services/android/overview.html) and [iOS SDK 4.x for Experience Cloud Solutions](https://docs.adobe.com/content/help/zh-Hans/mobile-services/ios/rel-notes.html).
